# MADO - 視野と反応の脳トレ：AI相談文

## このドキュメントについて
MADOアプリの技術面・内容面について、別のAIに相談するための包括的な資料です。
現状のスナップショットと課題を整理しています。

---

## アプリ概要

**MADO - 視野と反応の脳トレ**
UFOV（Useful Field of View）パラダイムに基づく、処理速度・注意力トレーニングアプリ。

- **対象ユーザー**: 認知機能の維持・向上に関心がある一般ユーザー（高齢者含む）
- **コアメカニクス**: 画面中央に一瞬表示される刺激（車/トラック）を識別する。正解が続くと表示時間が短くなり、間違えると長くなる（階段法）
- **収益モデル**: 無料（1日1セッション+広告）/ Pro ¥480買い切り（無制限+広告なし）
- **現在のフェーズ**: MVP（Task 1のみ実装済み）、App Store未申請

## 階段法（Staircase Method）の実装

```
アルゴリズム: 2-down / 1-up
- 2回連続正解 → 表示時間短縮（難しくなる）
- 1回不正解 → 表示時間延長（やさしくなる）

初期値: 333ms（Task 1）
ステップ幅: 50ms → リバーサル1回目以降 17ms（精密化）
終了条件: 9リバーサル
閾値算出: 最後の6リバーサルの平均値

フレーム制御: 60fpsで整数フレーム変換
最小: 1フレーム (16.7ms)
```

## 技術面の相談事項

### 1. Metal描画のタイミング精度
**現状**: CADisplayLink + フレームカウントで刺激提示時間を制御
**懸念**: 60fps前提だが実機は120fps。フレーム数とms変換にズレが出る可能性
**質問**: 研究レベルの精度が必要か？それとも一般向けアプリなら現在の精度で十分か？

### 2. AdMob SDK統合
**現状**: プレースホルダー実装。Google Mobile Ads SDKがSwift PMに対応しておらず、CocoaPodsが必要
**課題**: XcodeGenとCocoaPodsの併用は複雑。代替策として他の広告SDKも検討
**質問**: XcodeGen + AdMobの最適な統合方法は？AdMob以外のおすすめ広告SDKはあるか？

### 3. テスト戦略
**現状**: テストコード0行
**最低限必要なテスト**:
- StaircaseEngineのユニットテスト（閾値算出の正確性）
- DatabaseServiceの統合テスト
- SessionViewModelのロジックテスト
**質問**: MVP段階で審査通過に影響するテスト不足はあるか？

### 4. Task 2・3の設計
**Task 1**: 中央識別（実装済み）
**Task 2**: 分割注意（中央+周辺の同時判断）— Metal描画の拡張が必要
**Task 3**: 選択的注意（周辺に妨害刺激追加）— Metal描画+妨害ロジック追加
**質問**: Task 2・3のMetal実装アプローチ、UIフローの設計助言

### 5. データプライバシーとiCloud同期
**現状**: ローカルDBのみ（GRDB）。デバイス間同期なし
**将来**: iCloud同期でデバイス間データ共有
**質問**: GRDB→CloudKit移行 vs SwiftData+iCloud の選択。移行コストは？

## アプリ内容面の相談事項

### 6. UFOV研究の信頼性
**現状**: UFOV（Useful Field of View）パラダイムに基づく
**懸念**: 学術的に検証されたアルゴリズムを忠実に再現しているか不明
**質問**:
- 2-down/1-up階段法のパラメータ（初期値333ms、ステップ50ms→17ms、9リバーサル）は妥当か？
- UFOVの3タスク構造は正しく解釈できているか？
- 「閾値」をユーザーに「ウィンドウスコア」として見せる変換は適切か？

### 7. ターゲットユーザーと訴求
**現状**: 「視野と反応の脳トレ」というポジショニング
**課題**:
- 「脳トレ」市場は競合過多（Lumosity, Peak, Elevate等）
- UFOVの学術的裏付けは強み。だが一般ユーザーに伝わりにくい
**質問**:
- ターゲットを「高齢者の運転能力維持」に絞るべきか？それとも幅広く「脳トレ」でいくべきか？
- App Storeの説明文で学術的根拠をどこまで前面に出すべきか？
- 競合と差別化する最大のポイントは何か？

### 8. ゲーミフィケーション
**現状**: ストリーク（連続日数）、ウィンドウスコア、閾値トレンドチャート
**不足**:
- レベル/ランクシステムがない
- バッジ/アチーブメントがない
- 他ユーザーとの比較要素がない
**質問**:
- MVPでどこまでゲーミフィケーションを入れるべきか？
- リテンション改善に最も効果的な要素は何か？（ストリーク？レベル？バッジ？）

### 9. セッション設計
**現状**: 1セッション = 9リバーサルまで（約20-30試行、2-3分）
**懸念**:
- 無料ユーザーは1日1セッションのみ → 物足りない？
- セッション中に離脱した場合のデータ扱い
**質問**:
- 適切なセッション長は？研究では何試行/何分が推奨されるか？
- 無料ユーザーの制限（1日1セッション）は適切か？2-3回にすべきか？

### 10. オンボーディングの効果測定
**現状**: Duolingo風の4ステップチュートリアル（Welcome→Rules→3試行体験→結果表示）
**実装済み**: コーチマーク付き試行、スキップ可能、完了フラグ永続化
**質問**:
- オンボーディング完了率を計測する仕組みが必要か？
- 3試行の体験版は十分か？少なすぎるか？
- 結果画面でのAha moment（反応速度表示）は効果的か？
